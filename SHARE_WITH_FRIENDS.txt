Local AI Box — Windows quick start (Docker Desktop already installed)
================================================================================

DOWNLOAD
  https://github.com/tanvir-commits/AIBox/releases
  Get: privateai-box-vX.X.X-windows.zip  (or clone the GitHub repo)

YOU NEED
  - Windows 10/11
  - Docker Desktop installed AND running (whale icon — "Engine running")
  - This folder (full project, including docker-compose.yml)
  - ~8 GB free RAM for Docker Desktop (Settings → Resources → 8 GB+ if you can)
  - ~10 GB free disk for images

OPTION A — Double-click launcher (if PrivateAIBox.exe is in this folder)
  1. Start Docker Desktop; wait until it is running.
  2. Double-click PrivateAIBox.exe
  3. First run downloads images (often 5–15 min; no compile on your PC).
     Needs internet. Images: ghcr.io/tanvir-commits/aibox-backend + aibox-web
  4. Browser opens http://localhost:3000
  5. Login: admin@example.com  password: changeme
  6. Documents → upload a .txt or .pdf → wait for status "indexed"
  7. Chat → ask a question about that file
  8. Stop: double-click is not enough — run PrivateAIBox.exe -stop
     Or open PowerShell here: docker compose down

OPTION B — No .exe (batch file)
  1. Docker Desktop running
  2. Double-click scripts\start-windows.bat
  3. Same login and steps as above
  4. Stop: docker compose down

TROUBLESHOOT
  - "Docker not running" → open Docker Desktop, wait, try again
  - Port in use → close anything on ports 3000 or 8000
  - PC froze / "Interrupted" many times on pull → Docker was downloading 4 big
    images at once. Recovery:
      1. Reboot if needed, start Docker Desktop only (close other heavy apps)
      2. Open PowerShell in this folder
      3. docker compose down
      4. scripts\pull-images-safe.sh    (pulls one image at a time)
      5. docker compose up -d
      6. Open http://localhost:3000
    Low-RAM test mode: in .env set DEFAULT_EMBEDDING_PROVIDER=mock_embedding
    then docker compose up -d --force-recreate backend
  - Build errors → send your host the terminal output

OPTIONAL: real AI chat (not required for testing)
  - Install Ollama on Windows, pull a model
  - Edit .env: DEFAULT_LLM_PROVIDER=ollama
  - docker compose down && docker compose up -d --build
